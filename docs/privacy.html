プライバシーポリシー

最終更新日：2026年6月18日

Drive Path Copier（以下「本拡張機能」）は、ユーザーのプライバシーを最優先に設計されています。 本ポリシーは、本拡張機能が取り扱うデータと、その利用方法について説明します。

1. 取り扱うデータ

データ取得元目的保管場所
Google Drive のフォルダのメタデータ（フォルダ名、親フォルダ、共有ドライブ名） Google Drive API（読み取り専用） Windows 形式のパス文字列を組み立てるため 保存しない（処理後はクリップボードに出力するのみ）
Google アカウントのメールアドレス Google userinfo エンドポイント アカウントごとのドライブ文字設定を識別するため ブラウザのローカルストレージ（chrome.storage.sync）
ドライブ文字の設定（例：G、H） ユーザー入力 パス生成に使用 ブラウザのローカルストレージ（chrome.storage.sync）

2. データの外部送信について

本拡張機能は、いかなるユーザーデータも開発者や第三者のサーバーへ送信しません。 Google Drive API および userinfo エンドポイントへの通信は、ユーザー本人のデータを取得するために Google のサーバーと直接行われるものであり、それ以外の外部送信は一切ありません。 設定データ（メールアドレス・ドライブ文字）は、Chrome が管理するローカルストレージにのみ保存されます。

3. 取得する権限とスコープ

Google Drive API へのアクセスは読み取り専用に限定しています。要求する OAuth スコープは以下のとおりです。

  • .../auth/drive.readonly：フォルダ名・親フォルダ情報の取得（読み取り専用）
  • .../auth/userinfo.email：ドライブ文字設定のためのアカウント識別

ファイルの内容の閲覧・作成・編集・削除は一切行いません。

4. 第三者提供・販売

取得したデータを第三者に販売・譲渡・提供することはありません。広告・分析・トラッキングの目的での利用もありません。

5. データの削除

ローカルに保存される設定データは、拡張機能をアンインストールするか、設定画面から該当項目を削除することで いつでも消去できます。本拡張機能が外部に保持するデータは存在しません。

6. お問い合わせ

本ポリシーに関するお問い合わせは tanaka.yuki.dev@gmail.com までご連絡ください。

Privacy Policy

Last updated: June 18, 2026

Drive Path Copier ("the Extension") is designed with user privacy as the top priority. This policy explains what data the Extension handles and how it is used.

1. Data Handled

DataSourcePurposeStorage
Google Drive folder metadata (folder name, parents, shared drive name) Google Drive API (read-only) To build a Windows-style path string Not stored (output to clipboard only)
Google account email address Google userinfo endpoint To identify per-account drive-letter settings Browser local storage (chrome.storage.sync)
Drive-letter settings (e.g. G, H) User input Used for path generation Browser local storage (chrome.storage.sync)

2. No External Transmission

The Extension does not transmit any user data to the developer's or any third-party servers. Requests to the Google Drive API and userinfo endpoint are made directly to Google's servers solely to retrieve the user's own data. Settings (email and drive letters) are stored only in Chrome's local storage.

3. Permissions and Scopes

Access to the Google Drive API is strictly read-only. The requested OAuth scopes are:

  • .../auth/drive.readonly: retrieve folder name and parent information (read-only)
  • .../auth/userinfo.email: identify the account for drive-letter settings

The Extension never reads file contents, nor creates, edits, or deletes any files.

4. No Sharing or Sale

We never sell, transfer, or share collected data with third parties, and never use it for advertising, analytics, or tracking.

5. Data Deletion

Locally stored settings can be removed at any time by uninstalling the Extension or deleting the relevant entries from the settings screen. The Extension holds no data externally.

6. Contact

For questions about this policy, contact tanaka.yuki.dev@gmail.com.
