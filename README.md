# Drive Path Copier

Google Drive で開いているフォルダのパスを、Windows のエクスプローラー形式でクリップボードにコピーする Chrome 拡張機能です。

```
G:\共有ドライブ\プロジェクト\フォルダ名
```

## 機能

- 共有ドライブ・マイドライブ両対応
- Google アカウントごとにドライブ文字（G:、H:、I: など）を設定可能
- ポップアップ内の ⚙️ から設定画面を開いてアカウントとドライブ文字を紐付け

## インストール

Chrome Web Store からインストールしてください。（リンク準備中）

### 開発者向け（ソースから読み込む場合）

1. このリポジトリをクローンまたは ZIP でダウンロード
2. `manifest.json` の `oauth2.client_id` を自分の Google Cloud プロジェクトのクライアント ID に書き換える
3. Chrome で `chrome://extensions` を開き、デベロッパーモードを ON にする
4. 「パッケージ化されていない拡張機能を読み込む」でフォルダを選択

Google Cloud でのクライアント ID 取得方法は [セットアップ手順](docs/setup.md) を参照してください。

## 使い方

1. Google Drive でフォルダを開く
2. ツールバーの拡張機能アイコンをクリック
3. 「📋 Windowsパスをコピー」を押す
4. 初回のみ Google の認証画面が表示される → 「Allow」
5. クリップボードにパスがコピーされる

### ドライブ文字の設定

複数の Google アカウントを使い分けている場合、アカウントごとにドライブ文字を割り当てられます。

1. ポップアップ右上の ⚙️ をクリック
2. メールアドレスとドライブ文字（例：G）を入力して「追加」
3. 未設定のアカウントはデフォルトで `G:` が使われます

## プライバシーとセキュリティ

この拡張機能はユーザーのプライバシーを重視して設計されています。

### 収集・保存するデータ

| データ | 保管場所 | 目的 |
|---|---|---|
| Google アカウントのメールアドレス | ブラウザのローカルストレージ（`chrome.storage.sync`）| ドライブ文字の設定に使用 |
| ドライブ文字の設定（例：G、H） | ブラウザのローカルストレージ（`chrome.storage.sync`）| パス生成に使用 |

### 外部送信について

**いかなるデータも外部サーバーへ送信しません。**

メールアドレスを含むすべての設定データは、Chrome が管理するローカルストレージにのみ保存されます。`chrome.storage.sync` は Google アカウントに紐付いてデバイス間で同期されますが、この拡張機能自身のコード以外からはアクセスできません。

### API アクセスの範囲

Google Drive API へのアクセスは、フォルダ名と親フォルダ情報の取得（読み取り専用）のみです。ファイルの内容・作成・編集・削除は一切行いません。リクエストされる OAuth スコープは以下の通りです：

- `drive.readonly`：フォルダ情報の取得（読み取り専用）
- `userinfo.email`：ドライブ文字設定のためのアカウント識別

### ソースコードについて

このリポジトリで公開しているソースコードがそのまま動作しています。ビルドプロセスや難読化は行っていないため、動作内容をコードで確認できます。

## 技術仕様

- Manifest V3
- Google Drive API v3
- `chrome.identity` による OAuth 2.0 認証
- `chrome.storage.sync` による設定の保存・デバイス間同期
